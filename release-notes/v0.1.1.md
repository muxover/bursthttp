# bursthttp v0.1.1 — Proxy fixes and reliability

**Release date:** 2026-03-01

### Install

```bash
go get github.com/muxover/bursthttp@v0.1.1
```

---

## Summary

Patch release that fixes proxy behavior, error handling, chunked response parsing, and connection pool timing. Proxy timeouts no longer double on failure; slow or dead proxies no longer cause long cascading retries. Error classification (`IsTimeout`, `IsRetryable`) and CONNECT rejection messages are improved.

---

## What changed

### Proxy and connection behavior
- **Single-attempt dial** — Removed the 2-attempt retry inside `connectLocked` so proxy CONNECT no longer doubles effective timeout (e.g. 30s → 60s per new connection).
- **Faster failure with dead/slow proxies** — `GetConnection` now returns immediately after a failed `createConnection()` that took >1ms, instead of retrying up to 20 times (avoids 20×30s+ hangs).
- **Clearer CONNECT errors** — Proxy rejection now includes the HTTP status in the message (e.g. `proxy CONNECT rejected (HTTP 407)`).

### Correctness and RFC compliance
- **Chunked trailers** — `readChunkedBody` now drains all trailer headers after the terminal `0\r\n` chunk (RFC 7230 §4.1).
- **Nil request** — Nil request now returns `ErrInvalidRequest` (previously `ErrInvalidResponse`).
- **DoReader URL** — URL detection uses `strings.HasPrefix("http://")` / `strings.HasPrefix("https://")` instead of length-based checks.

### Error handling
- **IsTimeout** — Uses `errors.Is` / `errors.As`; correctly handles `ErrTimeout`, `context.DeadlineExceeded`, `*DetailedError`, and `net.Error.Timeout()`.
- **IsRetryable** — Uses `errors.Is` / `errors.As` for consistent unwrapping.

### Tests and benchmarks
- New tests: `TestProxyCONNECT407`, `TestIsForwardProxy`, `TestChunkedBodyWithTrailers`, `TestGracefulStopUnderLoad`, `TestMaxRequestsPerConn`, `TestErrInvalidRequest`, `TestIsTimeoutContextDeadline`, `TestIsTimeoutNetError`.
- New benchmarks: `BenchmarkWriteRequest`, `BenchmarkWriteRequestForwardProxy`, `BenchmarkReadResponse`, `BenchmarkViaHTTPProxy`.

### Other
- **ErrInvalidRequest** — New sentinel for nil request validation.
- **examples/basic** — User-Agent uses `bursthttp.GetVersion()` instead of hardcoded version.
- **ExampleMultipartBuilder** — Output check fixed for non-deterministic boundary (checks `len(body) > 0`, `len(contentType) > 0`).
- **streaming.go** — Removed unused `ReaderRequest` and `AddHeader`.
