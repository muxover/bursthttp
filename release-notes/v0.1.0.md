# bursthttp v0.1.0 — Initial release

**Release date:** 2026-03-01

### Install

```bash
go get github.com/muxover/bursthttp@v0.1.0
```

---

## Summary

First public release of bursthttp — a high-performance HTTP/1.1 client for Go with pipelining, per-host connection pooling, and zero external dependencies.

---

## What's included

### Core
- HTTP/1.1 client with persistent connection pooling and pipelining.
- Per-host connection pools with round-robin selection.
- Object pooling for requests and responses via `sync.Pool`.
- Fluent request builder API.
- Convenience methods: `Get`, `Post`, `Put`, `Patch`, `Delete`, `Head`, `Options`.
- URL routing with automatic host pool creation.
- Context cancellation and timeout support.

### Reliability
- Configurable retry with exponential backoff and jitter.
- Retryable status codes (429, 502, 503, 504).
- Connection rotation after N requests.
- Graceful shutdown with drain timeout.
- Idle connection eviction.
- Connection warm-up.

### Networking
- HTTP CONNECT proxy with authentication.
- SOCKS5 proxy (RFC 1928) with username/password auth.
- In-memory DNS cache with TTL.
- TLS with session caching and configurable handshake timeout.

### Data Handling
- Streaming response body as `io.ReadCloser`.
- Streaming request body from `io.Reader`.
- Multipart form-data builder.
- Gzip request compression and transparent response decompression.
- `Expect: 100-continue` support.
- Header injection protection (CRLF rejection).

### Observability
- Pluggable metrics interface with built-in implementation.
- Latency percentiles (p50, p95, p99), request/byte counters, pool events.
- Client stats snapshot.

### Configuration
- Presets: `DefaultConfig`, `HighThroughputConfig`, `ResilientConfig`.
