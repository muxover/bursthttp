# bursthttp v0.1.2 — Pre-encoded requests, lock-free pool, header zero-copy

**Release date:** 2026-03-11

### Install

```bash
go get github.com/muxover/bursthttp@v0.1.2
```

## Summary

Adds pre-encoded request headers, a lock-free connection pool path, and zero-copy header handling. You can cache the request header block and send many requests with the same headers without re-encoding; the connection pool uses lock-free structures on the get path; request and response headers support zero-copy where applicable.

## What's included

- **Pre-encoded requests:** `Request.PreEncodedHeaderPrefix` and `Client.BuildPreEncodedHeaderPrefix(req, host, port, useTLS)` to cache the request header block; only the body may change between sends.
- **Lock-free connection pool:** Host pool lookup uses `sync.Map`; per-host connection list uses `atomic.Pointer[[]*Connection]` with CAS for add/remove so the get path is lock-free.
- **Header zero-copy:** Request headers are sent via `net.Buffers` (vectored write); response header values via `Response.HeaderBytes(key)` returning a slice into the raw header buffer (valid until `ReleaseResponse`).
- Tests: `TestHeaderBytesZeroCopy`, `TestPreEncodedHeaderPrefix`.
