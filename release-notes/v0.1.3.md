# bursthttp v0.1.3 — Request scheduler, async DNS, connection health scoring

**Release date:** 2026-03-27

### Install

```bash
go get github.com/muxover/bursthttp@v0.1.3
```

---

## Summary

Phase 2 performance and control update. Adds a request scheduler that replaces spin-wait with a proper bounded queue, upgrades DNS resolution to be async with in-flight deduplication and background prefetch, and introduces per-connection health scoring so the pool prefers low-latency, low-error connections.

---

## What's included

### Request Scheduler
- New `Scheduler` type with per-host bounded queue and fixed worker goroutine pool.
- Replaces the 20×500µs spin-wait loop in `GetConnection` with a blocking channel queue — stable latency under overload, no busy-wait CPU burn.
- Workers call `pool.GetConnection` + `conn.Do` directly; full context cancellation at both enqueue and wait stages.
- Opt-in via `config.EnableScheduler = true`; enabled automatically in `HighThroughputConfig`.
- Configurable via `SchedulerWorkers` (default: `PoolSize`) and `SchedulerQueueDepth` (default: `workers×4`).

### Async DNS
- **In-flight deduplication:** concurrent misses for the same host share one `net.LookupHost` call (zero-dependency singleflight).
- **Background prefetch:** triggers a background refresh when 80% of TTL has elapsed — hot entries never expire on the critical path.
- **Round-robin IP selection:** atomic index cycles through all IPs returned by DNS on each lookup.
- **Stale fallback:** returns the last-known IP on transient DNS errors instead of failing the request.
- Janitor runs at TTL/4 intervals to catch the prefetch window accurately.

### Connection Health Scoring
- Per-connection **latency EWMA** (α=1/8) updated after every request.
- **Error rate** tracked over a rolling 50-request window.
- **Health score 0–100:** latency penalty (−1 per 10ms above 50ms, max −60) + error rate penalty (up to −40 for 100% errors).
- `getIdleConnection` scans up to 16 connections and returns the **highest-scoring** one instead of the first available; short-circuits on score=100.
- Gated by `config.EnableHealthScoring` (enabled by default).

### New config fields
| Field | Default | Description |
|---|---|---|
| `EnableHealthScoring` | `true` | Per-connection latency/error scoring |
| `EnableScheduler` | `false` | Route requests through per-host worker queues |
| `SchedulerWorkers` | `0` (= PoolSize) | Worker goroutines per host |
| `SchedulerQueueDepth` | `0` (= workers×4) | Max queued requests per host |
