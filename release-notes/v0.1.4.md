# bursthttp v0.1.4 — TCP tuning, pipeline auto-tuning, negative DNS cache

**Release date:** 2026-03-27

### Install

```bash
go get github.com/muxover/bursthttp@v0.1.4
```

---

## Summary

Three targeted improvements to how connections are established and how pipelining is managed. TCP Fast Open and SO_REUSEPORT reduce connection setup cost on Linux. Pipeline depth now adjusts automatically per connection based on real latency — fast servers get deeper pipelines, slow servers get shallower ones. And failed DNS lookups are now cached briefly so a bad hostname doesn't cause a DNS hammer on every request.

---

## What's included

### TCP socket options (Linux)
- **TCP Fast Open (`TCPFastOpen`)** — sends data on the first SYN packet, cutting one round-trip from connection setup on supported Linux kernels and servers.
- **SO_REUSEPORT (`TCPReusePort`)** — allows multiple sockets to bind the same address/port for better kernel load distribution.
- Both options are Linux-only (no-op on other platforms) and opt-in via config.

### Pipeline auto-tuning
- **`EnablePipelineAutoTune`** — when enabled, each connection independently adjusts its pipeline depth based on measured round-trip latency (EWMA from health scoring).
- Fast connections (<10ms) grow pipeline depth toward `MaxPipelinedRequests` (hard cap: 32).
- Slow connections (>100ms) shrink pipeline depth toward 1.
- 10–100ms range: depth held steady.
- Enabled by default in `HighThroughputConfig`.

### Negative DNS cache
- **`DNSNegativeTTL`** (default: 5s) — failed DNS lookups are cached for this duration. Repeated lookups for a bad hostname return immediately instead of blocking on `net.LookupHost`.
- Negative entries are cleared automatically when a subsequent lookup succeeds.
- Janitor evicts expired negative entries alongside expired positive entries.

### New config fields
| Field | Default | Description |
|---|---|---|
| `TCPFastOpen` | `false` | Linux: TCP Fast Open on new connections |
| `TCPReusePort` | `false` | Linux: SO_REUSEPORT on new sockets |
| `EnablePipelineAutoTune` | `false` | Dynamically adjust pipeline depth per connection |
| `DNSNegativeTTL` | `5s` | How long to cache failed DNS lookups |
